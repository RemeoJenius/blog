广度优先搜索

本章介绍图。首先介绍了图，之后介绍了第一种图的算法——广度优先搜索（breadth-first search,BFS）。
广度优先搜索让你能够找出两样东西之间的最短距离，不过最短距离的含义有很多！使用广度优先搜索可以：
  • 根据你的人际关系网络找到关系最近的医生等。

    图简介

    图是一个多对多的数据结构，由两个部分组成，第一部分：节点，第二部分：边。如果你从一个地方去往另一个地方，恰巧这个地方是你第一次去，别人只给了这个地方的地址。如果是我的话，我首先会打开某地图app去搜索，然后app会给我推荐几条线路，在线路上的选择我们往往会选择最短路径，这可能是前往该地点最短路径。解决最短路径的算法被称为广度优先搜索。
    要解决这个问题我们一般需要以下两个步骤：
    • 使用图来建立问题模型。
    • 使用广度优先搜索解决问题。

      广度优先搜索

      广度优先搜索是一种用于图的查找算法，可帮助回答两类问题：
  • 第一类问题：从节点A出发，有前往节点B的路径吗？

  • 第二类问题：从节点A出发，前往节点B的那条路径最近？

    假设，我急需用钱，我在脑中整理我的关系网络，先去找我的朋友借钱，假如我就有两个朋友姜某和于某（可见我的人缘并不怎么样）我先找于某借，他说昨天晚上跟她的对象出去了身上没有钱了，没办法我只能去找我的另一个朋友姜某，姜某说：“大哥，你还不了解我嘛，昨天买烟还是你跟你借的钱呢。”，我也很无奈，最后没有借到钱。当然我也可以去找他们两的朋友：马某和赵某借钱，从人际交往上来说，于某和姜某是我的朋友，我们这里认为是一度关系，于某和姜某的朋友，马某和赵某我们认为是二度关系。也就是说，朋友是一度关系，朋友的朋友是二度关系。
    具体是这样的，现在脑中构建一个借钱人的名单，先把我的朋友于某和姜某写在名单中，再去找于某借钱，如果于某没钱，把于某划掉，将于某的朋友马某写进名单中。再去找姜某，如果姜某没有钱再把姜某划掉，在将姜某的朋友写在名单中，然后不断的迭代如果找了名单中所有的人都没有钱，就意味着我借钱失败啦。这就是广度优先搜索。

    查找最短路径

    再说一次，广度优先搜索是一种用于图的查找算法，可帮助回答两类问题：

  • 第一类问题：从节点A出发，有前往节点B的路径吗？（我的人际网络里谁有钱借给我？）
  • 第二类问题：从节点A出发，前往节点B的那条路径最近？（在可以借钱给我的人当中谁离我最近）
    例如：朋友是一度关系，朋友的朋友是二度关系，以此类推。
    在你看来，一度关系胜过二度关系，二度关系胜过三度关系，以此类推。因此你应现在一度关系中搜索，确定其中朋友当中没有钱后，才在二度关系中搜索。广度优先搜索就是这样做的！在广度优先搜索的执行过程中，搜索范围从起点开始逐渐向外延伸，即先检查一度关系，在检查二度关系。因此这样查找下来，不仅查找到了你与朋友之间的路径，而且还是最短的路径。
